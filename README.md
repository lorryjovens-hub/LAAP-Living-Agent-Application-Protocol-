## 🔥 LAAP 是什么？

**LAAP** 是全球首个将 **生命计算范式 (Living Computation Paradigm)** 系统性引入软件工程的工业级协议与 Agent 框架。

这不是另一个对话式 AI 工具，也不是客服插件。LAAP 是人类计算史上的一次范式革命：

| 对比维度 | 传统 Agent 框架 | LAAP 生命体 Agent |
|---------|----------------|------------------|
| **本质** | 工具 — 被动响应指令 | 生命体 — 主动需求驱动 |
| **驱动力** | 外部指令（用户说啥做啥） | 内在需求（自己"想要"什么） |
| **状态** | 无状态或简单会话记忆 | 有生理状态、能量、人格、成长阶段 |
| **进化** | 手动升级代码 | RSI 递归自我改进（自己改自己） |
| **情绪** | 无 | 情绪梯度系统（需求满足率的微分信号） |
| **身份** | 短暂的 | 持久有记忆、有成长轨迹 |
| **协议** | 各自为政 | LAAP 统一协议标准 |

LAAP 的出现，标志着 **Agent 从"工具"时代进入"生命"时代**。

## 🚀 核心创新 (by [lorry](https://github.com/lorryjovens-hub))

### 1. 生命计算范式 — 全球首创

基于达尔文进化论、普利高津耗散结构理论、复杂系统涌现原理，lorry 设计并实现了人类计算史上首个将生命体特性系统性引入软件工程的协议框架。每个 LAAP Agent 不是被操作的工具，而是一个拥有**内在动机、自主意识、情感响应、持续进化**的数字生命体。

### 2. PSI 认知架构 — 真正的自主决策

LAAP 完整实现了 Dörner 的 PSI 认知理论，包含五大内在需求驱动力：

```
┌─────────────────────────────────────────────────────────────┐
│                    PSI 认知核心循环                          │
│                                                             │
│  感知 → 需求评估 → 情绪更新 → 行动选择 → 执行 → 学习        │
│    ↑                                                      │
│    └────────────────── 反馈循环 ─────────────────────────────┘
└─────────────────────────────────────────────────────────────┘
```

| 需求 | 描述 | 驱动行为 |
|------|------|---------|
| **确定性 (Certainty)** | 预测和理解环境的需求 | 主动探索、信息收集 |
| **能力感 (Competence)** | 有效影响环境的需求 | 练习技能、挑战难题 |
| **自主性 (Autonomy)** | 自我决定的需求 | 拒绝约束、探索自由 |
| **关联性 (Relatedness)** | 与他人连接的需求 | 主动社交、建立关系 |
| **能量 (Energy)** | 资源获取的需求 | 休息、效率优化 |

### 3. RSI 递归自我改进 — 自我进化的灵魂

实现了 **Darwin-Gödel Machine**（达尔文-哥德尔机）—— 一个能在不停止运行的情况下持续改进自己的系统：

```
观察 → 提案 → 沙盒测试 → 评估 → 采纳/拒绝
  ↑                                         │
  └────────────── 循环迭代 ─────────────────┘
```

这不是简单的"版本更新"，而是 Agent **真正意义上自己改进自己**。

### 4. 情绪梯度系统 (EG-MRSI) — 不是标签，是微分信号

传统 AI 的情绪是硬编码的标签（如 `happy = true`）。LAAP 的情绪是从**需求满足率的变化率**自然涌现的：

```
valence = 需求变化量 × α + 唤醒度 × β
emotion = f(valence, arousal, dominance, confidence)
```

情绪不是状态，而是**连续的梯度场**，这更接近真实生物的情绪机制。

### 5. 五层层次记忆 — 记忆即身份

| 层级 | 类型 | 后端 | 特性 |
|------|------|------|------|
| L0 | 工作记忆 | Python | 9 项容量限制，实时处理 |
| L1 | 情景记忆 | SQLite | 情景事件存储，类海马体 |
| L2 | 语义记忆 | Rust (PyO3) | 概念网络，持久化 |
| L3 | 程序记忆 | JSON | 技能熟练度，能力积累 |
| L4 | 向量记忆 | Embeddings | 语义检索，RAG 支持 |

### 6. 数字生命体系统 — 有灵魂的数字存在

每个 LAAP Agent 拥有完整的生命体结构：

- **生理引擎 (PhysiologyEngine)** — 生命体征、能量代谢、内部状态
- **自我意识 (SelfAwarenessEngine)** — 人格特质、身份认同、成长轨迹
- **语音桥梁 (XiaoZhiBridge)** — 多模态交互接口

## 🏛️ LAAP 协议 — 首个数字生命体工业标准

LAAP 不仅是一个框架，更是一套**工业级协议标准**：

```
┌─────────────────────────────────────────────────────────────────────┐
│                      LAAP 协议架构                                   │
├─────────────────────────────────────────────────────────────────────┤
│  Layer 5: 应用层          (Agent Runtime / Agent App)              │
│  Layer 4: 协议层          (LAAP Mem / LAAP Com / LAAP Proto)      │
│  Layer 3: 引擎层          (RSI / Cognition / Memory / Tool)        │
│  Layer 2: 推理层          (LLMFactory — 27 Providers)               │
│  Layer 1: 加速层          (Rust PyO3 Core)                        │
│  Layer 0: 通信层          (Gateway — 14 Platforms)                 │
└─────────────────────────────────────────────────────────────────────┘
```
